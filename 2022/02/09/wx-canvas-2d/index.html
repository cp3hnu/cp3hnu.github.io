# 微信小程序 Canvas 2D 绘制封面

接到一个任务，要求在微信小程序里绘制宣传封面，然后保存到用户的相册里，然后用户分享被朋友或者发朋友圈。本文将介绍两种微信小程序通过 canvas 绘制封面的方式，然后比较他们的不同点。

  • 原来的 CanvasContext (opens new window)
  • 新的 Canvas 2D (opens new window)

# CanvasContext

以前微信小程序通过 CanvasContext (opens new window) 来进行 canvas 绘制。具体操作如下：

微信小程序官方文档表示 CanvasContext 是旧版的接口，API 将不再维护，推荐使用新版的 Canvas 2D 接口

首先在 template 上定义一个 canvas 组件，在 canvas 组件上定义一个 canvas-id 的属性

本文的样例使用 uni-app 开发

<canvas style="width: 1920px; height: 1080px;" canvas-id="myCanvas"></canvas>
1

然后通过 createCanvasContext 方法获取 CanvasContext 对象

const context = wx.createCanvasContext('myCanvas', this)
1
2

最后就可以使用 CanvasContext (opens new window) 里的方法进行封面的绘制。这里封面将介绍绘制封面常用的两个方法，绘制图片和绘制文字

# 绘制图片

CanvasContext 使用 drawImage (opens new window) 方法绘制图片

// static 图片
context.drawImage('/static/images/xxx.png', x, y, width, height)
// 或者 本地图片
context.drawImage(tempFilePath, x, y, width, height)

drawImage 方法可以绘制两种类型的图片，一个是 static 目录下的图片资源，另一个是本地图片，包括本地临时文件。如果是网络图片需要先通过 getImageInfo (opens new window) 或者 downloadFile (opens new window) 下载到本地，然后进行绘制。

# 绘制文字

CanvasContext 使用 fillText (opens new window) 方法绘制文字

context.setTextAlign('left')
context.setTextBaseline('top')
context.setFontSize(15)
context.fillStyle = '#000000'
context.fillText('text', x, y)

uni-app 文档上说 CanvasContext 可以 设置 font (opens new window)，但是微信小程序是不支持的，所以 CanvasContext 不支持设置自定义字体

# 导出图片

微信小程序使用 canvasToTempFilePath (opens new window) 导出 canvas 绘制的图片到本地临时文件，但是这个方法只能在 draw (opens new window) 方法的回调中才能成功保存绘制的内容。 canvasToTempFilePath (opens new window) 通过 canvasId 找到对应的 canvas。

// false: 本次调用绘制之前 native 层会先清空画布再继续绘制
context.draw(false, () => {
  uni.canvasToTempFilePath({
    canvasId: 'myCanvas',
    success: (res) => {
      uni.saveImageToPhotosAlbum({
        filePath: res.tempFilePath,
      })
    }
  })
})

# Canvas 2D

微信小程序在 2.9.0 起支持一套新 Canvas 2D 接口，通过 Canvas.getContext('2d') 接口可以获取 CanvasRenderingContext2D (opens new window) 对象，实现了 HTML Canvas 2D Context 定义的属性、方法。

首先在 template 上定义一个 canvas 组件，在 canvas 组件上指定 type = 2d

  <canvas type="2d" id="myCanvas"></canvas>
1

然后通过 getContext('2d') (opens new window) 方法获取 CanvasRenderingContext2D 对象

onReady() {
 const query = uni.createSelectorQuery()
 query
	.select('#myCanvas')
	.fields({ node: true, size: true })
	.exec(res => {
		const canvas = res[0].node
		canvas.width = 1920
		canvas.height = 1080
		this.canvas = canvas
		this.context = canvas.getContext('2d')
	})
}

# 绘制图片

CanvasRenderingContext2D 使用 drawImage (opens new window) 方法绘制图片，但是这里绘制图片的方式跟 CanvasContext 完全不一样，不支持字符串的方式，只支持 CanvasImageSource (opens new window)，例如 HTMLImageElement (opens new window)。此外微信小程序提供一个 createImage (opens new window) 方法来创建CanvasRenderingContext2D 可以绘制的 Image 对象，我们可以使用这个方法来绘制 static 目录下的图片、本地图片以及网络图片

const src = ... // static 目录下的图片、本地图片以及网络图片
const image = await this.loadImage(src)
context.image(coverImage, x, y, width, height)

// 需要等图片加载成功之后才能绘制
loadImage(src) {
	const image = this.canvas.createImage()
	image.src = src
	return new Promise((resolve, reject) => {
		image.onload = () => {
			resolve(image)
		}
		image.onerror = error => {
			reject(error)
		}
	})
}

uni.loadFontFace({
  family: 'FZShangKJW',
  source: 'url("xxx.TTF")',
	global: true,
	scopes: ['native', 'webview']
})

# 导出图片

微信小程序使用 canvasToTempFilePath (opens new window) 导出 canvas 绘制的图片到本地临时文件， canvasToTempFilePath (opens new window) 通过 canvas 指定要保存的内容。

uni.canvasToTempFilePath({
  canvas: this.canvas,
  success: (res) => {
    uni.saveImageToPhotosAlbum({
      filePath: res.tempFilePath,
    })
  }
})
